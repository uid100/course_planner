These layers commonly map to service types or categories. Specifically Infrastructure-as-a-Service (IaaS), Software-as-a-Service (SaaS), Platform-as-a-Service (PaaS). Sometimes, collectively these cloud computing services are referred to as Anything-as-a-Service and abbreviated (XaaS).

Why is this important?

Understanding the different service models will drive engineering solution and architecture decisions. The different models are similar to the differences between a renting a car, riding on a transit bus, or using a ride-sharing service. All three solutions will carry a passenger from point to point, but all three have different planning and operational decisions.

With a rental car, you will need to be capable of operating the vehicle in the intended environment, and while your maintenance costs are minimal operating costs such as gas, tolls, and parking are your responsibility.

Using a city bus, you can join a planned route at particular points and ride between stops, but you will need to have knowledge of the route and any required transfers. The cost of usage is very low but the flexibility of destinations in most cases will be the most limited.

Ride-share services have gained popularity in recent years and are available in major cities worldwide. With this option, the user has great flexibility to specify origin and destination, the user avoids all maintenance and operating costs and in many cases may have less flexibility than the rental car, but for short distances, this can be a very affordable option.

Likewise, if your engineering solution requires a global internet available web application which relies on business data managed and stored in a database with access limited to authorized users then the solution might include building a database on a cloud-based database (PaaS) server, hosting a web application as another one or more web application or Application Programming Interface (API) (PaaS) services, and integrating with in Identification-as-a-Service (IDaaS) run by another provider as a Software-as-a-Service (SaaS). This multi-tiered solution may be running on the same or multiple cloud providers such as AWS or Microsoft Azure.

Alternatively, the same solution could involve employing a cloud-based virtual machine (vm) and installing and configuring database software, a web server, and writing software to authenticate user with password encryption and multi-factor authentication stored in a separate database. Using this Infrastructure-as-a-Service (IaaS) will give the engineering team the most control over every aspect of the installation, but require the most responsibility for ensuring all of the applications are patched to acceptable security levels and that all of the individual software licenses are maintained.
