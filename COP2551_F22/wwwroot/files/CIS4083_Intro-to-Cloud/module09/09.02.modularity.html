Separation of Concerns

Multi-layer, and n-Tiered design patterns are important in solution development and especially cloud computing.

Layers are used describe the logical separation of functionality, for instance the front end or presentation layer from the business logic, and the data access layer(s). Closely related to that is distributing those functional layers to phyisically separate components, such as a web server, api server, and database server.

To use the cake analogy, cakes may have layers inside them, and multiple individual cakes can be stacked in separate tiers.

    Why?
  • Security
  • Scalability
  • Maintainability
  • Extensibility

Service-Oriented Architecture

While not strictly the same as an n-Tiered application which has logical boundaries between process blocks, Service-Oriented Architecture (SOA) operates on a similar concept of decomposing individual actions and components to independent services. The concept of application layers, and tiers is similar but SOA uses self-contained services which may be shared among more than one external application.

A web application may store data in a SQL database running on a cloud server, and a separate mobile application may also access that same database, while an administrator is accessing the database and running queries from an RDBMS client on the server at the same time. Although each appication interacts with and relies on the database, the service itself is likely unaware of other processes and applications accessing it.

Advantages of running applications across multiple tiers or even discrete services, includes the ability to scale each component and even use load-balancing to share the processing across multiple resources.
