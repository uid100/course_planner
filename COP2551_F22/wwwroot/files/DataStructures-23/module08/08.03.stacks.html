Stacks

In C++, a stack is a linear data structure that follows the Last-In-First-Out (LIFO) principle. This means that the most recently added item is the first one to be removed. It can be thought of as a collection of elements with two main operations: push and pop. Elements are added to the top of the stack using the push operation, and they are removed from the top of the stack using the pop operation.

Operations and Characteristics

Push

The push operation is used to add an element to the top of the stack. It increases the stack's size by one. In C++, you can use the push() member function of the std::stack container, which is part of the C++ Standard Library. 

        #include 
            std::stack
                myStack;
                myStack.push(10);
                myStack.push(20);

Pop

The pop operation removes the element from the top of the stack. It decreases the stack's size by one. You can use the `pop()` member function to accomplish this. 

        myStack.pop(); // Removes the top element (20)

Top

You can retrieve the element at the top of the stack without removing it using the top() member function. 

        int topElement = myStack.top(); // Retrieves the top element (10)

Size

You can determine the number of elements in the stack using the size() member function. 

        int stackSize = myStack.size(); // Returns the size of the stack (1 in this case)

Empty

You can check if the stack is empty using the empty() member function. 

        bool isEmpty = myStack.empty(); // Returns false if the stack is not empty

Implementation

Stacks can be implemented using arrays or linked lists, but C++ provides a convenient stack container, std::stack, that is based on a deque (double-ended queue) by default.

Custom Data Types

Stacks can hold elements of custom data types, not just integers. You can create a stack of any user-defined class or struct. 

        #include 
        #include 

        std::stack stringStack;
        stringStack.push("Hello");
        stringStack.push("World");

Uses

The last-in-first-out data structure is useful in situations, when a solution might need to keep track of an order of operations that could be undone in the reverse order.
