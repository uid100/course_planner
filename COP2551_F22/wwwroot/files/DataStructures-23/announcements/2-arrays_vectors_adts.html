Abstraction is a cornerstone in programming and software design. It involves creating classes, functions, or modules that hide the inner workings of a system, providing a simplified interface for interaction. For example, in object-oriented programming, a class might represent a real-world object or concept, and its methods and properties abstract away the complex internal implementation details.

Hiding the complexities, or inner workings of objects and concepts simplifies their use and allows process and system design to be more effective and efficient.

In the first module, we looked at primitive or built-in data types and their operators. This week, we'll start to consider abstract data types (ADTs).
