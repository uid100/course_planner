Abstraction

Abstraction is the process of hiding the internal details of an application from the outer world. Abstraction is used to describe things in simple terms. It’s used to create a boundary between the application and the client programs.

Abstraction is one of the basics principles of object-oriented programming. Its definition is one that you should learn and practice for technical interviews. Be prepared to cite it and describe it in your own words.

An automobile, or a television, or a microwave oven are classic examples of abstractions. We use them. Most of us take their operation for granted, not needing to be aware of the fine details of the inner workings.

Classes and various collections in code are described as abstract data types. Consider the definitions for now, and we will visit them again in upcoming modules, and consider more examples.

An abstract data type (ADT) is a data type is a data type that specifies the values the data type can hold and the operations that can be done on them without the details of how the data type is implemented.

You've learned about classes and objects, instances of classes. Consider classes in the context of an abstract data type. The object can be passed between processes as a single cohesive unit with the encapsulated detail hidden within the class itself.

In this module, review and practice the use of classes and objects in C++, and the use of arrays and vectors. Try some of the examples in the textbook before diving into the project assignment.

We will look at more ADTs, and more details about different methods to use and manipulate them in the upcoming modules.
