Module 6

We have seen how searching and sorting algorithms have different complexity which is related to the size of the input data. This week, take another look at compound data types which are comprised of other types.

These data types will not be new to you, but consider them from the perspective of enumeration. If you have a disorganized collection of things in a pile, can you iterate through them in any logical way, or to process them, would it require simply a random selection? Which tire on your car is the first one? Does it make sense to iterate through collections with no apparent enumeration?

In this module, review and compare structs (from chapter 7) and the discussion of objects in chapter 11. How are they different? How are they similar?

Sorting Objects

For a coding exercise, read and store a set of data records, and using good object-oriented programming, sort them using different attributes.

In this module, you will:

  • Review structs and data objects.
  • Consider approaches to iterating and enumerating compound data types.

The goals for this week:

  • Apply sorting algorithm(s) to compound data types.
