Object-Oriented Programming (OOP) in C++

Objects and Classes

Objects are instances of classes

Classes define the blueprint or template for creating objects. They encapsulate data and behaviors (methods) into a single unit. Objects represent real-world entities and can interact with each other to model complex systems.

Instance Members

Instance members in a class are attributes and methods that belong to individual objects created from that class. These members are specific to each object and can have different values or states. For example: 


    class Car {
        public:
        std::string brand;
        int year;

        void StartEngine() {
            // Code to start the car's engine
        }
    };

In this example, brand and year are instance variables, and StartEngine() is an instance method. Each car object has its own brand and year values.

Static Members

Static members in a class are shared among all objects of that class. They are associated with the class itself rather than with individual objects. Static members are defined using the static keyword. For example: 


        class MathUtil {
            public:
                static double PI;

                static double CircleArea(double radius) {
                    return PI * radius * radius;
                }
        };

        double MathUtil::PI = 3.14159265359;

In this example, PI is a static member of the MathUtil class. It is shared among all instances of the class.

The 'this' Parameter

The 'this' pointer is a special pointer available in C++ methods that points to the object on which the method was invoked. It allows you to access the instance members within the class. For example: 


        class Person {
            public:
                std::string name;

                void SetName(std::string newName) {
                    this->name = newName; // 'this' is used to access the instance variable
                }
        };

Principles of Object-Oriented Programming

Object-Oriented Programming follows these key principles:

  • Encapsulation: Bundling data (attributes) and methods (behaviors) into a single unit (class) and controlling access to them.
  • Inheritance: Creating new classes (derived or child classes) from existing classes (base or parent classes) to inherit their attributes and behaviors.
  • Polymorphism: Allowing objects of different classes to be treated as objects of a common base class, enabling flexibility and extensibility.
  • Abstraction: Simplifying complex systems by representing the essential features while hiding unnecessary details.

These principles promote code reusability, maintainability, and modularity in software development.
