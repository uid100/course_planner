What is a web application?

As you already know, a webpage is a document that is formatted using HTML tags. You should already be familiar with what they look like and how to write them. Now imagine every time someone searches Amazon for "lunch box for back to school," or "licensed NFL jersey," or "western saddle," or whatever your last 5 Amazon searches contained. Each of those pages is an HTML formatted page which is dynamically created, just for the moment, and isn't stored on any server after it has been delivered. That is the difference between a web page and a web application.

You could probably write a visually compelling page containing a restaurant's logo, their address, and maybe even their menu choices, but a page result that contains the weather forecast for the next 3 days in any city would need to be generated by a program, using designed elements to make it visually appealing and useful.

 

.NET/C# and what?

You have already built applications using C# for .NET. ASP is Microsoft's Active Server Pages solution that will handle the details of the page transfer for we don't have to dive that deep into the protocols. We point Visual Studio to serve us a web page, and ASP does the rest for us.

 

MVC framework?

A framework is a pattern or a set of guidelines and best practices to follow which can make a product easier to build and maintain. If you walk through a house which is under construction, and the walls are just going up, you can use your imagination and picture what it's going to look like. You may not have the builder's vision for tile and color and landscaping, but you'll recognize interior doorways which are usually 78 inches high and even without anything installed, you can probably identify which rooms are intended to be bathrooms, bedrooms, kitchens, closets and so on. All this from just looking at the framework.

Using the Model-View-Controller design pattern, a developer will be able to quickly construct and maintain an application and ensure that it is properly constructed, and when it needs to be updated, you will know where to go to make the changes.

So what can you expect?

There are 16 weeks in the term. We're going to walk through Murach's ASP.NET Core MVC (2nd Edition!) textbook

This course was built using the previous edition of the book, and for an emerging technology, there was simply too much that needed updating. That's the reason for the new book edition. On the other hand, because it's a new edition, I have gone through the material to ensure content and references are updated, but if I've missed anything, let me know!

 

We're going to be using Microsoft Visual Studio. The current release is '2022.' The Community Edition is a free download. It will take some time and not an insignificant amount of storage space on your computer, but it is the industry standard and you will need this to be successful. I have a current release, and will be installing any updates this weekend. We're not working on legacy code, so there's no reason not to use the latest and find any sticking points as we go along. (Real-world stuff!).

Our textbook may reference different versions of the tool, and we may encounter some differences and even some challenges along the way, but that really is what we sometimes face as code developers. When you find these things, let us know and we'll work through them together. (Discord is great for that!)

 

Mostly, we'll be writing a lot of code. Set aside some time. Write out the examples in the book as you are reading. The time will be well spent.
