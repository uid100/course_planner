Project #1

So what is the objective?

The first thing is to make sure your computer is ready and configured to start writing code, building and hosting your web apps. Yes, this is important, but there's more.

We're sending you out to walk through the forest and look at all the things, and there's so much to see, but let me point to a couple of things for you to notice.

First, open it up, take a quick glance at all the files. Don't forget to unzip them before you try to open them. This sounds obvious, but modern operating systems will let you browse the contents of a zip file without actually unzipping it. And yes, it's something that I still do, pretty often.

In the root folder, you'll see a solution (.sln) file and a folder (directory) ....

*brakes squealing... wait, wait, wait!*

If you haven't already done this, make sure your File Explorer (Windows) or Finder (Mac) are set to (1) show file extensions and (2) hidden files. These are simple configuration steps. It's really important, but I'm going to leave it to you to look up the steps for your particular operating system.

The solution file (.sln) is a text file. Try opening it with your favorite text editor first to see what's in it.
without knowing anything about it, it looks like it has some configuration settings. "VisualStudioVersion" probably tells me what built it, Minimum-- probably how far back it will support, it's got a project and an equals with a "GuitarShop" string and a relative path to a .csproj file (we'll find out later that is a project file). There are what look to be opening and closing tags and a bunch of GUID- looking numbers. Nothing that looks like familiar C# code though. I suppose this is a configuration file that tells visual studio what to do and how to open files up.

I suppose I could navigate to that .csproj file and look inside that, but I'm getting impatient, so I'll wait until later.

Double click the solution file and Visual Studio launches (if it's installed) and the solution (with the project) opens up.

By default, the Solution Explorer is open. I like this tool and unless I'm really limited for screen space, I always have it open.

I usually do this on my Windows PC, but I like to code and test in both environments, so here's what it looks like on Visual Studio for Mac.

The file structure is important. At the top, there is a "solution." This is from that text file that we looked at. Next is a project. The solution may contain multiple projects, but for now, just the one is enough. This is described in that GuitarShop.csproj text file that we didn't look at yet.

Inside the Project Folder, there are a handful of folders and some files in the root folder. It looks like a random mess, but it's actually very intentionally organized.

Notice there are separate folders for Models, Views, and Controllers. These are plural, so they may contain many. We'll be discussing what goes inside each, but for now, they're kept carefully together.

In the project folder there are C Sharp (.cs) files, Program and Startup. More on these later, but this is how our program will be launched. There's a json file and a couple of ".cshtml" files. We know about .cs files and we know about .html files, but this is some kind of strange hybrid of the two. And yes, we'll talk more about them later.

Ok, let's start this thing!

Like any C# program, we need to compile and build it first. You can use the build menu to make sure it all builds without errors and watch the process. But you probably already know that if you just use the run or debug command, anything that needs to be built, will.

Here, the application is launched and running in my Google Chrome browser and if you look closely, you'll notice that it's running on my local computer on port 5001. This may be very different on another computer or on another configuration. Just notice the port number for now. You won't need it, but it's worth noticing. You can copy that URL and open it in another browser if you want to see what that looks like.

You can click through the menu items to see the different pages. Here is the product list, this time viewed on the Safari web browser. This image doesn't show it, but in Chrome it's easier to see that the URL is now "https://localhost:5001/Product/List"

We'll learn more about this later, but what it means is that we're using the "https://"  Hyper-Text Transfer Protocol (HTTP), over Secure Socket Layer (SSL), to the server at "localhost," on port 5001, and calling the Product controller and the List action method.

If it worked, then you have everything up and running and you're ready to keep going.
