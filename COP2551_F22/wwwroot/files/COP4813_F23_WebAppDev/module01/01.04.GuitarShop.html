Start Visual Studio and open the Guitar Shop app

Although there is no code for you to write in this exercise, take your time with it and notice some important points.

The Solution Explorer in Visual Studio is an important part of the environment and provides a lot of useful information at a glance.

At the top of the solution explorer, notice the name of the Solution. More about this later. Below that is the list of Projects. In the case of the Guitar Shop app, there is only one project in the solution, GuitarShop. Its name is in bold font. A solution may have several projects, but only one will be set as the default startup project. You can see this from the right-click (on the project name) context menu.

These instructions are specific to Windows. I will try and update for macOS as time allows.

For now, right-click on the solution name and Open Folder in File Explorer . Here you will see that there is a single solution (.sln) file. Right click it and open it in notepad. It is a small text file that visual studio uses to configure Visual Studio. It contains information about version numbers and which projects are in the solution (just one for now). Close that file. Note, ff you double-click on one of these .sln files, it will open the solution in Visual Studio. Since this solution is already open and running in Visual Studio, you won't need to do that.

Next, from the File Explorer, open up the GuitarShop folder and look at the list of files and folders there. Here you will see one Project (.csproj) file. Again open this file in a text editor (Right-click > Open with > Notepad). This is another small configuration, text file. Look at the settings, but don't worry about what they are for now, and close the file without editing it.

These configuration files are important to Visual Studio, but you will almost never need to edit them directly.

Close Notepad (if it's still running) and the File Explorer and return to Visual Studio.

Run the Guitar Shop app

Continue with the instructions in the exercise to launch the app. As described in the chapter, Visual Studio will start a hosting service and the app will open in a web browser.

The exercise steps point out some important points about the application. Read through each carefully.

From the browser, notice the URL. It probably says
https://localhost:##### where #### indicates a pseudo-random available port number the browser and host server are using.

This is actually a shortcut for the page, try adding
/Home/Index to the end of the URL. Also, as you click through each of the links in the app, notice how the URL changes. We'll learn more about these soon.

While we are here, try coping the URL and opening another browser and navigating to the same [localhost] address. You can open many sessions of this page, as long as the app is open. When they are all closed, the session will end. If you stop the app (in Visual Studio), the browser pages won't close, but if you try refreshing them when the app is no longer running, you will see an error.

Close the Guitar Shop app and exit Visual Studio

Build a New App from the Template

Reopen the Guitar Shop app

If you closed Visual Studio, reopen it. The Guitar Shop app will be at or near the top of your recent solutions. You can select it from there, or choose Open Project or Solution and navigate back to it.

Add a New Project to the Solution

From the Solution Explorer, right-click on the solution name near the top of the display, and select Add and New Project. From the New Project Dialog, you can use the filters to make it easier to find the template.

Change 'All languages' to 'C#'.

Change 'All project types' to 'Web'

Scroll down to find "ASP.NET Core Web App (Model-View-Controller)". There are other templates which look very similar. Be sure you choose the one with MVC included in the template. Click [Next].

By default the new project will be named "WebApplication1" If that project already exists, the id number will increment to a unique name. You can use the default, or rename the project to "WebApplicationMVCTest". Either is fine. Click [Next]

Check the Framework, the default should be .NET 6.0 (Long Term Support).

Click [Create] and notice that the new project is added to the solution. Right-click on the name of the project and select Set as Startup Project. Click the green triangle (play button) on the toolbar to build and start this new project in the debugger. Again it will launch the default browser and the hosted app will appear.

The MVC template includes a simple HomeController in the Controllers folder and a Home folder in the Views folder with an Index.cshtml file in it. There is more in the template, but it's enough for now that it runs. You will become very familiar with creating MVC web apps.

Submit the Assignment

Close Visual Studio and navigate to the folder that contains your solution (.sln) file, then open the folder above that one.

In Windows 11, use the File Explorer, for macOS, the Finder window. From the folder (directory) that contains your solution file, right-click the folder name and select Compress to ZIP file.

This may be a little different in Windows 10, but it works the same way.

From macOS, right-click the folder and choose Compress folder_name.

Upload your zip file, containing the solution with both projects. Remember this contains more than simply the .sln or .prj (or .csproj) files, as these do not contain the project code.

If you are having trouble Zipping files, Be sure Visual Studio is closed, as well as any applications that may be running in Visual Studio or as background processes and try again.

Do not use other archive utilities, such as tar, cpio, gzip, or 7zip, or others.
