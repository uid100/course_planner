Much of the course will be reinforcing common patterns and strategies, some will be new, and as with every new language, the way to succeed and even enjoy the experience is to immerse yourself in it.

So what can you expect?

There are 15 weeks in the term. We're going to walk through one of Tony Gaddis' Starting out with... textbooks.

If you don't already have it, find it quick. If you got a great deal on an older edition, you may be ok for at least a few weeks. We're going to be using the Eclipse IDE for Java. I have a current release, and will be installing any updates this weekend. We're not working on legacy code, so there's no reason not to use the latest and find any sticking points as we go along. (Real-world stuff!). We'll spend a few weeks migrating knowledge of object-oriented programming and familiar procedural design patterns to a different syntax and some new rules. We'll be learning some new libraries and approaches to graphical user interfaces (GUIs). Mostly, we'll be writing a lot of code. There are (online) exams every couple of weeks, and some projects to apply what you are reading about and practicing with sample code.

 

Speaking of online, the college lists this as a web course. You'll also hear it referred to as asynchronous. We won't have scheduled meetings, but we do have a class schedule to accomplish. The Canvas course is designed to guide you through, but we'll do it together, and I will make an effort to not let you feel alone through the process.
