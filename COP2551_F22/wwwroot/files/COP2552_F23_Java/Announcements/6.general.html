This is an advanced OOP class. Remember to use objects in your solutions. Start by asking what the entities are. 

 

Project 3 is an activity or exercise tracker. What are the objects? 

My solution has a menu object and exercise objects. You want to run? I create a run object and add it to a list. Weight lifting? create the object, add it to a list. Dance? Play Tennis? Go Bowling? Crete a "Dance" "Tennis" and "Bowling" object.  When the user requests a summary, simply iterate through the list and use the ToString() method from each created object.  You want to create 17 new exercises? simple. They all track almost the same thing, so they all inherit from the Exercise class. 

Remember the DRY (do not repeat yourself) principle.  If you have 3 exercises that all do almost the same thing, then move those lines of code to the base class and only keep the distinctions in the derived classes.
