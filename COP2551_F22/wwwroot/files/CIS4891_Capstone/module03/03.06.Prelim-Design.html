Project Assignment

Preliminary Design Documents

This week, migrate your project description from the last module into a GitHub repository and create and include these preliminary design document artifacts

  • Description
  • Wireframe diagram(s)
  • Solution architecture diagram
  • User Stories
  • Initial Use Cases
  • Use Case Diagram

 

GitHub

Create a public repository in GitHub. The name should be helpful in describing your application (not "Student_capstone_project" or "Project_1")

Add a brief (1-line) description for your application. You can edit this using the gear icon next to About on the front page of the repository.

Add some meaningful topical keywords. These can be relevant to the solution domain (e-commerce, healthcare, fitness, ...) or to the technology you are using (mvc, javascript, api, sql-server, ...)

Add a Readme (README.md) file. This file uses Markdown format and becomes the front page for your project as a place to collect and reference your design docs.

 

README.md

Your README file should be organized and contain logical headings such as Overview, User Stories, Use Cases

Copy and paste your project description from the last module, format as Markdown and edit as needed. It may be appropriate to add the document to a documents folder within the repository and only include select pieces of it, sufficient to describe your project to visitors.

Wireframe Sketches

Create wireframe concept drawings of your application to communicate an at-a-glance idea of your vision for the solution.

These diagrams are not expected to be polished or finished drawings. They should provide a logical view of the user's typical workflow from one screen to the next. Ideas and logic are important, artwork is not.

Use the drawing tool of your choice. My preference for this is the sketchy template in the free tool: Pencil Project from Evolus.

Draw.io is another good (free) choice.

Export your drawings as Jpeg images and save them to an images folder in your repository and link them to your README.
