Prototype

One definition is an early sample, model, or release of a product built to test a concept or process. Prototyping is an important component in Agile software design and development providing early realization or model of a concept.

Functionality is not required or expected, but rather the prototype model is useful for communicating and developing a concept for consideration and evolution.

A prototype may be constructed to meet one (or many of) the a project's top level requirements (from User Stories), with preference given to ideal scenario processing rather than edge-case and error handling.

For example:

User Stories

  • As a busy mobile professional,
  • I need the ability to securely transfer funds between bank accounts on-the-go
  • So that I don't need to drive to the bank during business hours and my accounts remain secure.

Requirements

requirements
ID Requirement
1. Provide secure access to bank accounts
1.1 The system shall enable a secure login capability
1.2 The system shall display a list of the customer's accounts
1.3 The system shall display the balance in each of the customer's accounts
2. Provide ability to transfer funds between available accounts
2.1 The system shall provide an interface for transferring available funds between accounts
2.1.1 The system provide a form to select the account from which to transfer funds, the target account to receive the transfer, and the amount to transfer
2.1.2 The transfer page shall limit the amount of the transfer to the amount available in the account
2.1.3 The system shall reduce the displayed available balance by the amount of the transfer after it is confirmed

The prototype interface should include each of the pages (forms) needed to complete the transaction and be appropriate and adequate to display the functionality. External interfaces are not required or expected to be functional. Data should be within expected values, but may be temporary values substituted for live data.

Submission

For this step, submit the URL for the github project repository and a comment indicating 'ready for review.' Within the repository make a folder called prototype and a (separate) readme, which includes a narrative with screenshot images or links to HTML documents outlining the functionality of the prototype.

Sketches or mockups are acceptable at this stage. If you are comfortable creating HTML code at this point, it will be a helpful headstart for later.

It's important to contain the prototype in a separate (or sub-) folder to clearly identify this as a prototype with no expetation for finished functionality.
