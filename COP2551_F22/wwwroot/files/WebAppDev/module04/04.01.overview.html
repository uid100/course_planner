Data-driven design

In chapter 2, we learned how to create a strongly-typed view. if this concept is not clear to you, it would be worth a few minutes to look that up to refresh the idea. In the Model-View-Controller framework, the data model is the focus. The View is a representation of that data model, the controller handles the requests and facilitates moving the data in and out of the Response view.

The Data drives the application. It is the reason for the web request, the content of the response. A well-designed data model can make the application coding much simpler.

In Chapter 4, we will use Entity Framework to design a data structure and migrate it to database tables.
