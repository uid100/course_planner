Chapter Notes

Decomposing a Problem

In chapter 2, we are going to dig deeper into the topic of application development. One of the key things to look for in the chapter is decomposing a problem. If we were tasked with coding a financial system, there isn't anybody (instructor included) that could just sit down and start writing the code for it. A project of this scale requires a lot of analysis and design work before any of us could write any useful code for the project.

Decompose big problems into
smaller, easier to understand pieces.

Once we break the financial system into sub-systems or components such as accounts payable, accounts receivable, general ledger, etc., the problem becomes easier to understand. We may need to break these smaller problems into even smaller problems yet before we can fully understand what is needed and develop a solution.

A subtle but key topic of this chapter is the first of 3 fundamental coding structures Sequential code where one statement executes after the other.
